## 🚀 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### API Endpoint: `/api/families`

**Метод:** GET  
**Кеш:** 30 секунд  
**Лимит:** 10 семей

**SQL запрос:**
```sql
SELECT 
  m.user_id_1, m.user_id_2,
  u1.first_name AS f1, u1.username AS u1,
  u2.first_name AS f2, u2.username AS u2,
  m.married_at,
  EXTRACT(DAY FROM NOW() - m.married_at) AS days
FROM marriages m
JOIN users u1 ON u1.user_id = m.user_id_1
JOIN users u2 ON u2.user_id = m.user_id_2
WHERE m.divorced_at IS NULL
ORDER BY m.married_at ASC
LIMIT 10
```

**Ответ:**
```json
[
  {
    "rank": 1,
    "user1Id": 12345,
    "user1Name": "Иван",
    "user2Id": 67890,
    "user2Name": "Мария",
    "marriedAt": "2026-04-01T12:00:00Z",
    "days": 35
  }
]
```

### Функция плюрализации: `formatMessages()`

**Использование:**
```typescript
formatMessages(1)    // "1 сообщение"
formatMessages(2)    // "2 сообщения"
formatMessages(5)    // "5 сообщений"
formatMessages(21)   // "21 сообщение"
formatMessages(111)  // "111 сообщений"
```

**Правила:**
- 1, 21, 31, ... → "сообщение"
- 2-4, 22-24, 32-34, ... → "сообщения"
- 0, 5-20, 25-30, ... → "сообщений"
- 11-14 всегда → "сообщений"

## 📝 ЗАМЕТКИ

### TypeScript ошибки в IDE:
Красные подчёркивания в VS Code — это временные ошибки TypeScript сервера. Они не влияют на работу приложения. Код корректен и будет работать при сборке.

**Причина:** VS Code иногда теряет связь с типами React/JSX.

**Решение:** Перезапустить TypeScript сервер или VS Code (опционально).

### База данных:
Все новые функции работают с той же PostgreSQL базой, что и бот. При отсутствии подключения к БД компоненты gracefully скрываются, не ломая страницу.
