### 2. 👤 Профили игроков (ИЗ АУДИТА)

**Проблема:** Нельзя посмотреть статистику конкретного игрока

**Решение:**
- Страница `/profile/[user_id]` или `/u/[username]`
- API `/api/profile/[user_id]`
- Карточка игрока с полной статистикой

**Что показывать:**
- Титул (по total_earned)
- Место в топе
- Баланс и всего заработано
- Дуэли (победы/поражения)
- Клады найдено
- Достижения (X из 30)
- Серия фермы (текущая и рекорд)
- Брак (если есть)
- Прогресс до следующего титула
- График активности

**Файлы:**
- `app/profile/[id]/page.tsx`
- `app/api/profile/[id]/route.ts`
- `components/profile/player-card.tsx`
- `components/profile/stats-chart.tsx`

**SQL:**
```sql
SELECT 
  user_id, username, first_name,
  balance, total_earned, total_spent,
  farm_streak, max_farm_streak,
  duels_won, duels_lost,
  treasures_found,
  pidor_count,
  farm_success_count,
  casino_games_count,
  created_at
FROM users
WHERE user_id = $1
```

**Эффект:**
- ✅ Геймификация (хочется улучшить свой профиль)
- ✅ Социальный аспект (сравнение с друзьями)
- ✅ Вирусность (делятся профилями)

### 3. 💍 Рейтинг семей (ИЗ АУДИТА)

**Проблема:** Браки есть в боте, но не видны на сайте

**Решение:**
- Топ-10 самых долгих браков
- API `/api/families`
- Компонент на странице `/live`

**Что показывать:**
- Пары (имена обоих)
- Длительность брака (дни)
- Эмодзи 💍 для активных браков
- Место в рейтинге

**Файлы:**
- `app/api/families/route.ts`
- `components/live/families-top.tsx`

**SQL:**
```sql
SELECT 
  m.user_id_1, m.user_id_2, m.married_at,
  u1.first_name AS f1, u1.username AS u1,
  u2.first_name AS f2, u2.username AS u2
FROM marriages m
JOIN users u1 ON u1.user_id = m.user_id_1
JOIN users u2 ON u2.user_id = m.user_id_2
WHERE m.divorced_at IS NULL
ORDER BY m.married_at ASC
LIMIT 10
```

**Эффект:**
- ✅ Мотивация создавать браки
- ✅ Социальная активность
- ✅ Контент для обсуждений
