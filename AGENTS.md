# AGENTS.md

## Что это

Сайт и админ-панель экосистемы «Возня» — Next.js App Router (Tailwind v4, pnpm).
Спутник Telegram-бота `voznya-bot`. Это **не** статический лендинг: приложение
читает PostgreSQL бота, имеет авторизацию (Telegram-логин + OIDC), сессии и
админ-панель с RBAC.

См. `README.md` для карты страниц и слоёв данных.

## Границы (важно)

**Бот = авторитет экономики и источник истины. Сайт = одобренный слой
взаимодействия.** Сайт МОЖЕТ выполнять одобренные транзакционные действия —
но только через аудируемые server-actions/admin-actions, которые соблюдают те
же правила экономики и гарантии владения, что и бот.

- PostgreSQL бота — источник истины по балансам и игровому состоянию.
- Любая запись с сайта обязана проходить через серверный слой (server-only
  `lib/*-actions.ts` или `app/api/**`), где: владелец берётся из подписанной
  сессии (`getSession().uid`), а не из тела запроса; действие атомарно (один
  `withTransaction`), под нужными `FOR UPDATE`; пишется аудит/леджер
  (`transactions`, `purchase_history`, и т.п.); валидации экономики (цена,
  остаток, баланс) повторяют логику бота.
- Админ-запись — через `app/api/admin/*` под RBAC. Логин — через OIDC-флоу.

**Разрешённые записи** (примеры): покупка подарков/Premium, депозиты, вывод
(future), действия с инвентарём (продать/подарить/вывести), авторские действия
админки, экономически-контролируемые действия.

**Запрещённые записи** (всегда): обход валидации экономики; обход проверки
владения; обход аудита/леджера; произвольная прямая правка состояния игрока
(балансов, инвентаря) в обход серверного слоя и его гарантий.

> Якорь-реализация: `lib/shop-actions.ts:buyGift` — порт `buy_gift()` бота:
> ownership из сессии, атомарная транзакция под `FOR UPDATE`, леджер
> `transactions` + `purchase_history`, та же валидация. Новые записи равняться
> на него.

## Команды

| Действие | Команда |
|---|---|
| Установка | `pnpm install` |
| Dev | `pnpm dev` → http://localhost:3000 |
| Сборка | `pnpm build` |
| Прод | `pnpm start` (после сборки) |

## Особенности

- **Package manager:** pnpm (`pnpm-lock.yaml` — единственный лок-файл).
- **Images:** `next.config.mjs` ставит `images.unoptimized: true`.
- **Типы при сборке:** `next build` пропускает валидацию типов (см. `next.config.mjs`).
- **Analytics:** `@vercel/analytics` грузится только в продакшен-сборке.
- **Тесты:** автотестов в репозитории нет — проверять сборкой + ручным просмотром.
- **MMR-ранги** продублированы с ботом (`lib/mmr.ts` ↔ `app/settings/mmr.py`) —
  менять синхронно.
